**Дисципліна:** Розробка мобільних застосунків

## Опис проєкту

Метою роботи є опанування навичок роботи з базовими компонентами React Native (View, Text, Image, FlatList, TextInput) та налаштування навігації у середовищі Expo. Застосунок має три основні екрани, об'єднані верхньою таб-навігацією.

### Основні функції:

  * **Головна:** Відображення списку новин за допомогою компонента `FlatList`.
  * **Фотогалерея:** Відображення сітки зображень (2 стовпці).
  * **Профіль:** Форма реєстрації з валідацією типів даних для введення (email, пароль, ПІБ).
  * **Навігація:** Використано `@react-navigation/material-top-tabs` для перемикання між екранами.

## Інструкція із запуску

1.  **Клонування репозиторію:**
    ```bash
    git clone https://github.com/OleksandrNahal/MobileLabsRN2026.git
    cd lab1
    ```
2.  **Встановлення залежностей:**
    ```bash
    npm install --legacy-peer-deps
    ```
3.  **Запуск Metro Bundler:**
    ```bash
    npx expo start --tunnel
    ```
4.  **Тестування:** Відскануйте QR-код за допомогою додатка **Expo Go** на Android або камери на iOS.

## Технічні особливості та вирішення проблем

Під час виконання роботи було розв'язано низку технічних проблем, пов'язаних із сумісністю SDK та нативною платформою Android:

1.  **Конфлікт типів Java:** Виправлено помилку `java.lang.String cannot be cast to java.lang.Boolean`, яка виникала при неявному перетворенні типів у пропсах `secureTextEntry` та `headerTitleAlign`. Рішенням стало явне приведення до `boolean` за допомогою тернарних операторів.
2.  **Проблеми Bundling:** Усунено помилку `Unable to resolve "@react-native/virtualized-lists"` шляхом примусового встановлення відсутніх нативних модулів та очищення кешу Metro (`expo start -c`).
3.  **Peer Dependencies:** Використано прапорець `--legacy-peer-deps` для вирішення конфлікту версій між `@types/react` та внутрішніми залежностями Expo SDK 54/55.
4.  **Типізація (TypeScript):** Додано інтерфейси `NewsItem` та `PhotoItem` для усунення помилок `implicitly has an 'any' type` у компонентах `FlatList`.

## Способи запуску та тестування

  * **Expo Go (Фізичний пристрій):** Використовувався як основний метод для перевірки нативної поведінки компонентів та жестів.
  * **Емулятор Android (WebStorm/Android Studio):** Дозволив проводити швидке налагодження інтерфейсу без підключення кабелів.
  * **Expo Tunnel:** Необхідний для роботи через складні мережеві конфігурації (наприклад, університетський Wi-Fi), що забезпечує доступ до локального сервера через зовнішню адресу [cite: 16-20].
